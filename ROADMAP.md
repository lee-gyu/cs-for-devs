# 개발자를 위한 CS 지식 로드맵

> 5년차 이상 경력 소프트웨어 엔지니어들이 **내부 동작 원리와 트레이드오프를 근거로 설계 판단을 내릴 수 있는 엔지니어**로 성장시키기 위한 심화(deep dive) 커리큘럼입니다.
> 모든 교육 문서는 한국어로 작성하며, 이 저장소의 `docs/` 디렉터리에 챕터별 배치합니다.

---

## 1. 과정 개요

| 항목 | 내용 |
|------|------|
| 대상 | 5년차 이상 경력 개발자 |
| 목표 | 컴퓨터 과학의 내부 시스템 구조를 이해하고 소프트웨어 개발에 필요한 설계 판단 및 디버깅을 스스로 할 수 있는 엔지니어 지식을 전달 |
| 산출물 | 챕터별 컴퓨터 과학 학습 자료, 실습 과제, 성능·구조 분석 리포트 |

### 학습 원칙

1. **원리 우선** — API나 도구 사용법이 아니라 그 아래의 동작 모델(자료구조, 런타임, 커널, 프로토콜, 스토리지 엔진)을 먼저 세운다. 사용법은 모델의 표현으로 익힌다.
2. **트레이드오프 중심** — 모든 기술 선택에는 비용이 있다. "무엇을 쓸까"가 아니라 "이 상황에서 각 선택이 무엇을 얻고 무엇을 포기하는가"를 판단 기준으로 삼는다.
3. **경계 조건 탐색** — 추상화가 무너지는 지점(성능 급락, 이론적 불가능성, 스펙의 한계)을 의도적으로 찾아가며 학습한다.
4. **표준과 1차 자료 중심** — RFC, 국제 표준, POSIX, 원 논문, 각 언어·도구의 공식 문서를 1차 자료로 삼고, 통념과 1차 자료가 다르면 1차 자료로 검증한다.
5. **만들며 검증하기** — 모든 챕터는 실습 과제로 마무리하되, "돌아간다"에서 멈추지 않고 벤치마크, 프로파일링, 패킷 캡처, 실행 계획 분석 같은 계측으로 왜 그렇게 동작하는지까지 확인한다.

## 2. 전체 커리큘럼 구조

모든 챕터는 10분~15분 분량의 `00-introduction.md`로 문제의식과 학습 지도를 먼저 세우며, 본문 문서는 45분~90분 분량을 기준으로 한다.

## 3. 챕터별 상세 커리큘럼

### 챕터 0 — CS를 배우는 이유

경력 개발자가 익숙한 도구의 사용법을 넘어 추상화 아래의 비용·동시성·실패 조건을 판단해야 하는 이유를 세운다. 이 파트는 전체 과정의 오리엔테이션이며, 이후 과목 목록이 아니라 현장 문제를 해석하고 검증하는 연결된 판단 지도로 사용하게 한다.

#### 오리엔테이션 (`docs/ch-0/`)

**학습 목표**

- CS 지식이 필요한 순간을 추상화의 의미·자원·실패 계약과 연결해 설명한다.
- 하나의 현장 문제를 여러 시스템 계층의 비용 모델·불변식·계약·증거로 분석한다.
- 현상에서 모델과 예측을 만들고 구현·계측으로 검증하는 학습 루프를 수행한다.
- 자신의 지식 공백과 업무 문제에 맞춰 순차 학습과 문제 중심 학습 경로를 선택한다.

| # | 문서 | 주요 내용 |
|---|------|----------|
| 0 | `docs/ch-0/00-introduction.md` | 경력 개발자에게 CS가 필요한 이유, 세가지 질문, 전체 커리큘럼으로 진입하는 두 학습 경로 |
| 1 | `docs/ch-0/01-when-abstractions-break.md` | 의미·자원·실패 계약, 규모·동시성·분산·실패에서 추상화의 경계가 드러나는 사례, 필요한 내부 깊이를 정하는 경계 지도 |
| 2 | `docs/ch-0/02-cs-in-practice.md` | 주문 확정 API 사례로 연결하는 알고리즘·런타임·OS·네트워크·DB, 비용 모델·불변식·계약·증거를 이용한 설계와 디버깅 |

**적용 과제**: 최근 경험한 장애나 설계 논쟁 하나를 골라 기대와 관찰의 차이, 계층별 경쟁 가설, 지켜야 할 불변식, 가설을 구분할 증거를 경계 지도로 정리한다. 관련 챕터 하나를 선택해 2주 학습 스프린트와 완료 기준을 작성한다.

### Part 1 — 이론 컴퓨터 과학

계산이라는 행위 자체의 모델과 한계를 세운다. 여기서 세운 형식 언어·오토마타·복잡도 모델은 Part 2의 컴파일러와 타입 시스템, Part 5의 검색·마이닝이 위임받아 사용한다.

#### 챕터 1 — 자료구조와 알고리즘 (`docs/ch-1/`)

**학습 목표**

- 점근 표기가 보장하는 것과 보장하지 않는 것(상수 인자, 캐시 효과)을 구분하고 실측으로 확인한다.
- 주요 자료구조가 메모리에 놓이는 방식을 근거로 상황에 맞는 구조를 선택한다.
- 실무 문제를 그래프로 모델링하고 표준 알고리즘으로 환원한다.

데이터를 모델링하고, 대규모로 처리하고, 의미를 찾아내는 과정을 다룬다. 저장 엔진 내부는 챕터 11, 분산 원리는 챕터 10을 기초로 삼는다.

#### 챕터 16 — 데이터 모델링과 데이터베이스 활용 (`docs/ch-16/`)

**학습 목표**

- 정규화의 근거를 이해하고 비정규화가 정당화되는 조건을 판단한다.
- 실행 계획을 읽고 인덱스 설계와 쿼리 구조로 성능 문제를 해결한다.

| # | 문서 | 주요 내용 |
|---|------|----------|
| 0 | `docs/ch-16/00-introduction.md` | 데이터 모델과 쿼리가 도메인 가정·정합성·성능을 함께 결정하는 이유, 스키마에서 실행 계획까지 이어지는 학습 지도 |
| 1 | `docs/ch-16/01-data-modeling.md` | 관계형 모델과 정규화의 근거, 비정규화 판단, 문서·그래프 모델과의 비교 |
| 2 | `docs/ch-16/02-query-processing.md` | 옵티마이저와 실행 계획 읽기, 인덱스 설계, 조인 전략과 N+1 (엔진 내부는 챕터 11 참조) |

**실습 과제**: 하나의 도메인을 스키마로 설계하고, EXPLAIN 실행 계획을 근거로 인덱스 튜닝 전후 성능을 비교하는 리포트를 작성한다.

#### 챕터 17 — 대규모 데이터 처리 (`docs/ch-17/`)

**학습 목표**

- 셔플 비용과 데이터 포맷을 근거로 배치 파이프라인의 병목을 진단한다.
- 이벤트 시간·워터마크·정확히 한 번 의미론으로 스트림 처리의 보장 수준을 판단한다.

| # | 문서 | 주요 내용 |
|---|------|----------|
| 0 | `docs/ch-17/00-introduction.md` | 데이터가 한 머신을 넘을 때 계산보다 이동·시간·재처리 비용이 중요해지는 이유, 배치와 스트림의 보장을 비교하는 학습 지도 |
| 1 | `docs/ch-17/01-batch-processing.md` | MapReduce 계보, 셔플의 비용, 컬럼너 포맷과 데이터 레이크 |
| 2 | `docs/ch-17/02-stream-processing.md` | 이벤트 시간과 워터마크, 정확히 한 번 의미론, 스트림·테이블 이중성 (분산 원리는 챕터 10 참조) |

**실습 과제**: 미니 배치 파이프라인(분할·병렬 집계)과 스트림 파이프라인을 구현해 처리량과 지연 시간을 측정하고 병목을 분석한다.

#### 챕터 18 — 검색·마이닝·지식 표현 (`docs/ch-18/`)

**학습 목표**

- 역색인과 랭킹 함수의 구조로 검색 품질 문제를 진단한다.
- 마이닝 결과의 과적합·검증 문제를 식별하고 신뢰할 수 있는 결론의 조건을 판단한다.

| # | 문서 | 주요 내용 |
