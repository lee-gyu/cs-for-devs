# CS for Devs

5년차 이상 소프트웨어 엔지니어가 컴퓨터 과학의 내부 동작 원리와 트레이드오프를 이해하고, 이를 설계·디버깅 판단에 활용할 수 있도록 돕는 한국어 심화 학습 자료입니다.

API와 도구의 사용법보다 그 아래의 자료구조, 런타임, 운영체제, 프로토콜, 스토리지 엔진을 먼저 살펴봅니다. 각 주제는 이론에 그치지 않고 구현, 벤치마크, 프로파일링, 패킷 캡처, 실행 계획 분석 등의 실습으로 검증합니다.

## 커리큘럼

전체 과정은 5개 Part, 18개 챕터, 66편의 문서로 구성됩니다. Part 순서는 이론에서 응용으로 이어지는 권장 학습 순서이며, 필요한 Part부터 독립적으로 학습할 수도 있습니다.

| Part | 챕터 | 주요 주제 |
| --- | --- | --- |
| 1. 이론 컴퓨터 과학 | 1–3 | 자료구조와 알고리즘, 계산 이론, 정보와 암호 |
| 2. 프로그래밍 언어와 컴파일러 | 4–6 | 언어 이론과 타입 시스템, 컴파일러와 인터프리터, 런타임과 메모리 |
| 3. 컴퓨터 시스템 | 7–12 | 컴퓨터 구조, 운영체제, 네트워크, 분산 시스템, 데이터베이스 내부, 가상화와 클라우드 |
| 4. 소프트웨어 공학 | 13–15 | 요구사항과 설계, 품질과 신뢰성, 프로세스와 배포 |
| 5. 데이터와 정보 시스템 | 16–18 | 데이터 모델링과 데이터베이스 활용, 대규모 데이터 처리, 검색·마이닝·지식 표현 |

각 챕터는 10–15분 분량의 인트로, 45–90분 분량의 본문 여러 편, 실습 과제로 구성되며, 기술 선택의 이점과 비용, 추상화가 무너지는 경계 조건, 표준과 1차 자료에 근거한 검증을 중요하게 다룹니다.
